The CPU stands for Central Processing Unit. It is often referred to as the 'brains of the computer', processing instructions using the FDE (Fetch Decode Execute) Cycle.

FETCH

1. The CPU uses the address bus to find memory location of the data/instrcutions to be fetched from main memory.

2. The data bus moves data/instructions from main memory to the MDR

3. The MDR copies this into the CIR

DECODE

4. The data bus moves the data/instructions to the CU

5. The CU decodes the instruction (looking at the instruction set)

EXECUTE

6. The instruction is executed

There are 4 main types of processor architecture you need to know...

  • von Neumann
  • Harvard
  • RISC processor
  • CISC processor
  • Co-Processors

von Neumann architecture

The von Neumann architecture makes use of the stored program concept.

Stored Program Concept Machine code instructions are fetched and executed serially by a processor that performs arithmetic and logical operations

In the von Neumann architecture, both data and instruction shared the same data bus and memory. This enables programs to be optimised in size, using memory efficiently as well as being able to manufacture it cheaper.

Harvard architecture

In the Harvard architecture, parallel data buses and instructions may be used, to improve performance.

However, a downside to this is that circuitry is more complex, longer and more expensive to manufacture.

As a result, programs tend to be longer and memory is often wasted.

RISC architecture

RISC architrecture has a smaller instruction set of 30-40 instructions (as opposed to CISC). These instructions are simpler in nature and therefore the computer has to do more work, taking up more RAM.

However, these instructions only take up one FDE cycle and therefore can make use of pipelining, to become as fast as CISC

CISC architectue

CISC processors have a large instruction set, with instrructions complex in nature

This emables less work for the computer and therefore less RAM is used.

However, instructions may take more than one FDE cycle and as a result, CISC processors cannot process from benefit from pipelining.

What is a multicore system?

A multicore syustem is where a processor has more than one core. The multiple cores on a processor eachj have their own level 1 cache which is stored on that cores and share level 2 cahce which is stored on the processor. Tasks are coordinated by a central task manager.

Primary Memory is directly accesible by the CPU (on the motherbaord). This includes RAM, ROM and Virtual Memory.

Secondary Memory is not directly accessible by the CPU (external to the motherboard. THis includes the Hardrive and SSD - for example.

RAM holds instructions (applications and user data) currently in use by the CPU, as well as the CPU as it is in use by the computer system. It is volatile as it holds data as small charges of electricity in tiny transistors which need to be refreshed every few milliseconds.

ROM is a non-volatile, special memory chip used to store the BIOS and load the operating system.

Virtual memory is hard drive space accessed when RAM becomes full. It operates a lot more slowly than RAM and is often used by the operating system as a lot of programs need to be loaded.

Memory Management

Memory Managment splits up main memory into sections to optimise the running of programs; this is done by paging and segmentation. Paging is the splitting of memory into fixed sized units (pages) of around 4kiB which is tracked using a memory logic table. Segementation, on the other hand, is the logical splitting of memory into varying length depending on its content. This fills up memory less efficiently than Paging.

Device Drivers

What is a Device Driver?

Device Drivers enable the OS and other software to able to interact with hardware devices at an abstracted level - without having to concern itself with the implementation of the device. The device driver extends the functionality of the operating system.

What is a Device Routine?

A program within the device driver which causes the device to perform specific tasks/receive requests from the device that requires the CPU to be interrupted.

Virtual Machines

What is a Virtual Machine?

A host system that runs software that enables other software to run as if they were running on a hardware system.

What are the Uses of a Virtual Machine?

  • Running other operating systems on a single computer
  • To run intermediate code
  • To run incompatible software from architectures that are no longer supported/unavailable (i.e. retro arcade games)
  • To create a test system

What is Intermediate Code?

Half-way between source code and machine code which is useful in reducing architecture dependency of applications/software. For example Java ByteCode is a Java intermediate code which is run on a Java VM.

  • From a Karnaugh Map, we can simplify expressions
